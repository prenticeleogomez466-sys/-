# novel-writer · 小说大模型(可执行封装)

> 把"小说大模型"(`D:\novels\升级` 100模块/10层方法论)的硬规则,加上从全网网文 skill 提炼的**可运行机制**,封装成一套照着跑的流程。核心升级:**把文笔规则从清单变成能跑的脚本**(扫AI味、测文风),并补上原系统最大的洞——**长篇连续性(防遗忘/防幻觉)**。

## 何时用
任何"写小说/写正文/写网文/爽文/出立项包/写细纲/写开篇/扩章/续写/去AI味"的任务。批量生成(>3部)走 Workflow + 对抗审查(见下「批量」)。

## 不可违背的硬规则(写之前先内化)
1. **口味=玄幻热血系统爽流**:废柴/低位觉醒 + 系统金手指当场见效 + 装逼打脸 + 越级碾压 + 章末大钩,短句砸节奏。文艺慢热克制=不合格。
2. **开篇钩子前置**:第一句甩最强钩子;前100字抛悬念或代入爽,**不准用风景/天气/背景交代开头**;每篇手法不同。
3. **文笔七条**:画面入场 / 感官落地 / 卡节奏(长短句砸) / 克制不说破 / 换俗比喻 / 对白带潜台词 / 禁潦草陈述。
4. **禁AI腔禁模仿**:见 `references/去AI味扫描表.md`,定稿前**必须跑** `scripts/deslop_scan.py` 扫一遍。不模仿辰东/番茄等名作腔调,每作要有从人物身份长出的专属声音。

## 标准流程(单部立项包 / 开篇)
按 L0→L9 走,但用下面的可执行件落地:

1. **选题(L1)** — 两步:**先定题材,再撞创意,最后套配方**。
   - **题材丰富,别默认玄幻**:先在 `references/题材矩阵.md` 选题材簇(玄幻/仙侠/都市异能/末世/科幻/无限流/规则怪谈/职业流/女频…),保留爽流内核、换题材皮。10 部别都玄幻退婚吞噬。
   - **撞创意**:新意来自远域相撞(法务流=律师×契约,bug猎手=程序员×规则),用 `references/创意杂交引擎.md`(题材 × 现实职业/学科 × 机制),可跑 `python scripts/idea_forge.py` 批量撞候选。
   - **配机制**:从 `references/金手指机制图谱.md` 18 大家族抽,避开用滥家族,过天花板/代价自检。
   - **套黄金配方**:**金手指 × 身份 × 处境 × 反差**。铁律:金手指越强处境越惨(反比);必须直接撬动眼前的局;反差写到读者眼前。配方表见 `references/文风锚定与爽点引擎.md`。批量时按**题材簇×远域元素双维分桶**防撞车(比纯机制分桶题材更丰富)。
2. **(可选)对标锚定(L5)** — 若用户给了对标书,跑文风锚定 SOP(`references/文风锚定与爽点引擎.md` + `scripts/style_stats.py`),产出可量化的句长/标点/潜台词特征,写作时贴着它走 = 最强去AI味。
3. **设定 + 10章细纲(L2/L4)** — 设定写金手指机制+天花板+成长线、世界观、主角小传、核心矛盾、卖点对标平台。细纲每章给:本章事件 + 爽点/打脸节拍 + **章末钩**(类型从 `references/钩子工具箱.md` 13式里选,**连续两章不用同一种**)。
4. **第一章正文(L5/L6/L7)** — 开篇钩子前置;黄金三章爽点前置;装逼打脸按节奏表(铺垫>打脸,前置小无敌,打脸对象要有分量);短句砸节奏;章末留约100字大钩。
5. **去AI味自检(L0 M00)** — **强制**:`python scripts/deslop_scan.py <文件>`,命中★★★★★(如"不是A而是B")即重度,逐条改;再人工过文笔七条。
6. **落盘** — 按类型+日期建子文件夹归类(不扔桌面根目录)。

## 长篇连载(防遗忘/防幻觉)——原系统最大补强
写超过单章/要连载时,**用文件系统当记忆,对话只负责创作**。见 `references/长篇状态追踪.md`:
- 大纲完成后建 `追踪/角色状态.md`(角色快照)+ `追踪/伏笔表.md`(叙事债务)+ `追踪/时间线.md`。
- **写每章前**先筛"最简记忆包"(只保留"不知道就会写错"的角色状态/伏笔成因/世界约束),不要全量加载。
- **写每章后**更新角色状态快照 + 勾掉/推进伏笔。
- 这套机制让连载几十万字不崩人设、不漏伏笔、力量体系不自相矛盾。

## 批量生成(>3部)——对抗审查
走 Workflow:① 多架构师按**不同金手指机制分桶**产配方(预喂已写书名防撞车);② 每部写立项包;③ **毒舌主编**评级(缜密/吸引/爆火三项,出爆款潜力股/可救/平庸/枪毙);④ 不达爆款潜力股按致命伤改写一轮;⑤ **读者+编辑评审团 2/3** 复核;⑥ `deslop_scan.py` 批量终检 + Grep 查坏链/格式垃圾 + 查同质化。详见 `references/钩子工具箱.md` 的爽点公式与 `D:\novels\升级` 的 L9 工业化模块。
