# 🔴 永久铁律·最高指令 —— 绝不兜底 / 绝不用不可信或缺失数据

> 2026-06-06，用户用本模型**投真钱**，定为**永久铁律、最高指令**，优先级高于一切默认行为：
> "以后永远如此 没有兜底 更不能用不可信 缺失的信息内容 所有制作都是 永久铁律最高指令"
> 并要求"按照我说的永远刻入到足球大模型里"（= 代码强制，非口头）。

## 铁律五条

1. **绝不兜底**：数据/能力不足时，不准用默认值/中性值/估计值/模板凑一个温吞输出顶上。
   该缺标"缺失·未参与判断"，该不推就 `unpredictable`（不推荐），宁可少给不可假给。
2. **绝不用不可信数据**：明显异常 / 未正则化 / 薄样本高方差拟合出的**垃圾值**比兜底更糟，
   必须识别并**剔除或标注**，不准当真值用或展示。
3. **绝不用缺失信息冒充**：`model-derived fallback`、`matches:0`、空数据层 = 缺失，
   不准说成"近期状态/排名/风格已纳入"。
4. **每个数字可追溯**：答不出"哪次运行、哪个源、什么数据"= 假，不准发。
5. **诚实认能力边界**：免费公开数据+模型在 1X2 上**系统性打不过收盘赔率**（本仓库回测实证），
   不准吹"深度研究出能稳赢方向"。模型价值 = 把市场读准 + 老实标哪场有把握/哪场抛硬币 + 对冲纪律。

## 代码强制点（carved-in enforcement）

| 铁律 | 代码位置 | 行为 |

| 不兜底拟合 | `dixon-coles-engine.js` | 样本不足 → `usable:false` 不退联赛先验/中性系数凑数（2026-05-31 已立） |
| 占位假盘 | `odds-authenticity.js` + `prediction-engine` scoreFromMarket/halfFullFromMarket | 主客对称的占位/模板赔率 → 丢弃回退 DC 真矩阵 |

| 缺失如实标 | `daily-report.js` 数据缺失·未预测 sheet / 各列⚠️闸 | 抓不到的标缺，不填默认 |

## 自检（所有产出前逐条过）

出任何"推荐/分析/报告/代码产出"前：① 有没有兜底默认值？② 有没有用垃圾/极端/未正则化值？
③ 有没有拿空数据/fallback 冒充真实？④ 每个数字能追溯到真实来源吗？⑤ 有没有吹不该吹的把握？
任一中招 → 先剔除或标缺，再产出。
