基准日：2026-05-15（Asia/Shanghai）。

以后生成正式足球推荐时，一律以这一天跑通的完整度作为最低标准；达不到标准就不应输出正式推荐。

## 最低门槛

- 实时闸门必须通过，并且闸门年龄不超过 `SOURCE_GATE_MAX_AGE_MINUTES`，默认 30 分钟。
- 官方赛程必须来自本次中国官方公开源同步。
- 当日所有官方场次必须 100% 覆盖：`usable === fixtures`、`complete === fixtures`、`missing === 0`。
- 当日所有市场快照必须 100% 实时：`marketRealtime === fixtures`。
- 竞彩足球每场必须具备：胜平负欧赔、让球胜平负、外部亚洲盘口。
- 14 场胜负彩必须识别完整 14 场；每场必须具备：欧赔、亚洲盘口。14 场本身没有官方让球胜平负玩法，不强行要求该层。
- 推荐审计必须 0 错误、0 警告；比分和半全场必须在胜平负结果确定后生成，不能冲突。
- 模型结构审计必须 0 错误。
- 正式推荐禁止使用 `--allow-missing-odds` 或旧的凌晨缓存来绕过数据缺口。

## 固化检查命令

```powershell
npm run standard:check -- --date=YYYY-MM-DD
```

这个命令会同时检查实时闸门、赔率覆盖、推荐一致性和模型结构；失败时应先补齐数据源，再生成推荐。

## 阶段能力审计

- 使用 `npm run model:stage-audit -- --date=YYYY-MM-DD` 逐段审计赛程、赔率、高级数据、预测、派生玩法、资金风控、输出自动化。
- 任一阶段出现 P0 时禁止正式推荐；必须先修复缺口并重新跑实时闸门。
- 该审计不替代 `standard:check`，而是用于定位哪个阶段拖垮模型能力。
