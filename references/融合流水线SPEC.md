```
①市场锚定 → ②题材+创意 → ③配方+机制 → ④设定+长篇骨架 → ⑤细纲
→ ⑥正文 → ⑦三层去AI+留存 → ⑧对抗审查+同质化 → ⑨落盘归档
```

## 三层质检(⑦,这是 v2 的硬关卡,缺一不可)
1. **词级**(确定性):`python scripts/deslop_scan.py 稿子` —— 退出码2(★5或★4×2)= 重度,逐条改到≤1。
2. **节奏级**(确定性):`python scripts/style_stats.py 稿子` —— 句长方差<6=匀速流水句必改;短句<55%/对白<25% 调节奏。
3. **结构级**(LLM-judge):`结构级判官.md` 8查(降智反派/套路桥段/角色同腔/升华收尾)+ 逐章留存弃书风险。`structAiScore<7` 或前三章 `retentionRisk=高` → 打回。
> 三层全绿才算定稿。这套把"去AI味"从一遍扫词,升级成 词级+节奏+结构 三道闸。

## 单部精简版(写一部立项包/开篇)
②→③→④(设定)→⑤→⑥(第一章)→⑦三层→⑨。市场锚①和长篇骨架按需。

## 批量版(>3部,Workflow 全量)
- 选题:架构师按 **题材簇×远域元素双维分桶**(`创意杂交引擎.md`)产配方,预喂已写书名防撞车。
- 写作 pipeline:写→`deslop_scan`+`style_stats`双脚本门→`结构级判官`→不过则改写一轮。
- 复核:毒舌主编+读者+编辑 2/3。
- 终检:`similarity_check.py` 测全集撞车 + Grep 查坏链/格式 + 各脚本批量过。
- 落盘:Word+评审xlsx+手机下载页,传GitHub。

## 连载验证版(证明长篇机制,#2)
选一部 → ④建追踪三件 → 循环{筛最简记忆包 → 写本章 → 三层质检 → 更新角色状态/伏笔表} → 每5章查力量体系/伏笔一致性。这是检验"几十万字不崩"的唯一方式。

## 一句话
**市场锚 + 题材丰富 + 远域杂交**(创意广度)× **黄金配方+爽点引擎+钩子**(爽感)× **三层去AI+长篇追踪+对抗审查**(质量)= 小说大模型 v2。
