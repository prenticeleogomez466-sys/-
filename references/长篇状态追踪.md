# 长篇状态追踪(防遗忘/防幻觉)——原系统最大补强

> 核心思想:**用文件系统当记忆,对话只负责创作**。连载几十万字不崩人设、不漏伏笔、力量体系不自相矛盾。短篇/单章不需要;连载或要续写时必用。

## 三个追踪文件(大纲完成后建,每章写完更新)
```
{项目}/追踪/
├── 角色状态.md   # 主要角色当前快照(最简记忆包的数据源)
├── 伏笔表.md     # 叙事债务:埋了什么、第几章埋、何时回收
└── 时间线.md     # 关键事件的时间顺序,防时序矛盾
```

## 写每章前:筛"最简记忆包"
不要全量加载历史。只保留 **"不知道这个,本章就会写错"** 的信息:
1. **角色状态**:本章细纲提到的角色 → 取其当前能力/关系/公众形象
2. **历史因果**:本章有伏笔回收 → 取埋设细节 + 前文铺垫
3. **世界约束**:本章涉及的力量体系/社会规则/地理限制

输出格式:
```
## 最简记忆包(第N章)
### 角色状态
{角色A}:{一句话当前状态,含最近变化}
### 相关伏笔/前史
{伏笔1}:{埋设细节,第X章},本章需要{回收/推进}
### 世界约束
{约束1}:{与本章相关的规则}
```

## 角色状态快照格式(`追踪/角色状态.md`)
```markdown
## {角色名}
- 当前身份:{最新身份/职业}
- 当前能力:{最新境界/能力水平}
- 关键关系:
  - 与{B}:{当前关系}(第N章变化)
- 公众形象:{外界如何看他}
- 待回收伏笔:{与他相关的未回收伏笔}
- 状态变更记录:
  - 第N章:{变化描述}
```
**更新规则**:每章写完检查身份/能力/关系/公众形象有无变化,有则更新对应字段+追加一行变更记录。只追踪主要角色(出场≥3次或有独立线)。每角色保留最近10条变更,超出则把最早的摘要进字段后删行。

## 伏笔表格式(`追踪/伏笔表.md`)
```markdown
| 伏笔 | 埋设章 | 内容 | 计划回收章 | 状态 |
|---|---|---|---|---|
| 玉佩来历 | 第3章 | 母亲临终给的旧玉佩,发烫 | 第30章左右 | 未回收 |
```
**叙事债务原则**:每埋一个钩就记一笔债;写每章前扫一遍"有没有到期该还的债";大高潮前清掉主要旧债,别烂尾。

## 力量体系/天花板一致性自检(防最常见崩盘)
立项时就把金手指**天花板和代价**写死,连载时每章对照:
- 金手指这次的用法,有没有突破立项定的天花板?
- 代价机制(折寿/反噬/冷却)这章兑现了吗,还是又被悄悄忽略?
- 越级碾压有没有把后面的战力空间提前掏空?(50部评审里最高频的致命伤就是"天花板第一章就摊满")
