# 扫榜与市场锚定 —— 选题贴当下热度,不靠静态记忆

> 问题:静态 playbook(起点近十年年冠)会过时。选题前**抓当下热榜**,知道现在什么题材/金手指在爆,把选题锚到实时趋势上。默认用 Playwright Chrome 自己抓(免费,不等用户截图)。

## 抓什么
| 站 | 榜单 | 看什么 |
|---|---|---|
| 起点中文网 | 月票榜/新书榜/分类榜 | 哪个题材簇在前列、新书冒头的金手指 |
| 番茄小说 | 飙升榜/完读榜 | 短平快爽点、下沉市场口味 |
| 七猫/晋江 | 分类榜(含女频) | 女频题材风向 |
| 知乎盐言 | 短篇热榜 | 短篇/短剧情绪点 |

## 抓法(Playwright,已验证免费路)
1. 用系统 Chrome 的 Playwright MCP(见记忆 project_playwright_mcp),navigate 到榜单页。
2. snapshot/evaluate 提取:书名 + 分类标签 + 简介一句。
3. 聚合:① 题材簇频次分布(现在哪个簇占榜多)② 高频金手指关键词 ③ 新书榜里"冒头的新机制"。
4. 输出 `市场锚.md`:本周热门题材 Top5 + 高频金手指 + 蓝海缺口(榜上少见但有潜力的)。

## 怎么用进选题
- **顺势**:热门题材簇 → 用 `创意杂交引擎.md` 加远域元素做差异化(蹭热度但不撞车)。
- **逆势找蓝海**:榜上几乎没有的题材簇(如规则怪谈/职业流)= 竞争小,适合做护城河,但要扛"爽点更难直给"。
- **避坑**:已霸榜烂大街的金手指(签到/吞噬…)直接进 `金手指机制图谱.md` 黑名单。

## 注意
- 反爬挡 Node 时走浏览器(Playwright 已验证可行);报 SSL 错清空 proxy 环境变量。

- 抓不到时降级用静态 `D:\novels\升级` + reference_hot_novel_playbook,别空等。
